Overview

What is a Single Answer Question?

A Single Answer question (also known as radio button or single-select) allows survey respondents to select exactly one option from a list of predefined choices. This is the most fundamental and widely-used question type across all industries.

  • Radio button interface - Visual indicator of single selection
  • Mutually exclusive options - Only one choice allowed
  • Universal support - Works across all 20+ platforms
  • 100% conversion accuracy - Perfect structure preservation

When you create a single answer question in Pirai AI using the MC:SingleAnswer type, it converts seamlessly to the native single-select format across Qualtrics, REDCap, SurveyMonkey, Google Forms, and all other supported platforms.

Key Formatting Rules

  1. First Row: Contains Block Name, Question ID, Question Text, MC:SingleAnswer, first option, and recode value 1
  2. Subsequent Rows: Same Block Name and Question ID, but leave Question Text and Question Type empty
  3. Answer Options: Each option gets its own row with sequential recode values (1, 2, 3, 4...)

Best Practices

‚úÖ Design Guidelines

Option Count

  • Minimum: 2 options
  • Optimal: 3-7 options for usability
  • Maximum: No technical limit, but 15+ options reduce response quality

Option Text

  • Keep concise (under 100 characters per option)
  • Use parallel structure ("Very satisfied" / "Somewhat satisfied" not "Very satisfied" / "I am somewhat satisfied")
  • Order logically (ascending/descending scales, alphabetical, or by frequency)

Required Options

  • Include "Other (please specify)" when list isn't exhaustive
  • Add "None of the above" when applicable
  • Consider "Prefer not to answer" for sensitive questions

Scale Design

  • Use 5-point scales for general attitudes (Strongly Disagree ‚Üí Strongly Agree)
  • Use 7-point scales for more nuanced measurement
  • Use 0-10 for NPS and recommendation likelihood
  • Maintain scale consistency across similar questions

‚úÖ Data Quality Tips

üí° Improve Response Quality

  1. Avoid Leading Questions: "Don't you agree that..." ‚Üí "To what extent do you agree..."
  2. Balance Options: Equal positive and negative options
  3. Include Neutral: Add middle option for true midpoint (e.g., "Neither agree nor disagree")
  4. Mutually Exclusive: Ensure options don't overlap
  5. Test Order Effects: Randomize when possible (set in platform after conversion)

FAQs

Q: Can I use images as answer options?

A: Pirai AI focuses on text-based specifications. After conversion to your target platform (Qualtrics, QuestionPro, etc.), you can add images using the platform's native editor.

Q: How do I make a question required?

A: Required/optional settings are configured in the target platform after conversion. Pirai AI handles question structure, not validation rules.

Q: Can I add skip logic based on answer?

A: Display/skip logic is platform-specific and should be added after conversion in your target platform's logic editor.

Q: What if my options have different recode values like 1, 2, 5, 10?

A: Non-sequential values are supported! Use any integer values that match your analytical needs (e.g., age ranges coded as midpoints: 22, 27, 32, etc.).

Q: How do I handle "Other (please specify)" with text box?

A: Include "Other" as a standard option in Pirai AI. After conversion, use the platform's native "Other with text box" feature if available.
