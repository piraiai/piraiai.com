The Format Problem Nobody Talks About

Every survey platform speaks a different language. Qualtrics wants a QSF file: a JSON structure that defines every question, block, and display logic element. REDCap wants a Data Dictionary CSV with precise column headers and pipe-separated choice coding. Google Forms has no import button at all; the only automated path in is through a JavaScript Apps Script. SurveyMonkey wants formatted plain text pasted into a specific field. Microsoft Forms accepts Word documents but only if they are structured a certain way.

None of them want your Word document as it is.

The document is not the problem. The format is. And for most research teams, the conversion step is invisible. Not because it does not exist, but because nobody has solved it until now.

Supported Platforms and Output Format Reference

Complete reference of every platform Pirai AI supports and the output format generated for each.

Qualtrics
QSF (.qsf): Qualtrics Survey Format containing all questions, blocks, and survey structure. Import via Survey tab > Tools > Import Survey. Also supports Qualtrics Advanced Format (.txt) for appending questions to existing surveys.
REDCap
Data Dictionary CSV (.csv): all required columns including Variable Name, Form Name, Field Type, Field Label, Choices, and validation settings. Upload via Project Setup > Data Dictionary > Upload File > Commit Changes.
Google Forms
Google Apps Script (.txt): a complete, ready-to-run script that builds the form when pasted into the Apps Script editor (three-dot menu > Apps Script in Google Forms). No coding required.
SurveyMonkey
Paste-ready text (.txt): formatted for SurveyMonkey's Paste Your Content importer. Copy and paste via Create Survey > Paste Your Content > Add Questions.
Microsoft Forms
Word document (.docx): structured for Quick Import at forms.office.com. Upload via Quick Import > Upload from this device. Choose Form or Quiz. Complete Start Review to finalise.
QuestionPro
Word document (.docx): formatted for QuestionPro's Add from MS Word feature. Import into a blank survey via the Add Question menu.
Zoho Survey
Plain text (.txt): structured for Zoho Survey's Import Document feature. Import via Create Survey > Import Survey > Import Document. Also compatible with Bulk Question Builder for existing surveys.
Alchemer
bConvert syntax (.txt): formatted with Alchemer's specific import syntax including () for radio buttons and ::NewPage:: for page breaks. Paste via Create Survey > Import from Word > Start Building.
Forsta
Three output formats available: bConvert .txt, Word .docx, or Forsta XML .xml. Import via Download Center > Import Survey. XML requires Forsta Plus v18 or later.
SurveySparrow
Bulk question text (.txt): formatted for SurveySparrow's Bulk Add Questions auto-detection. Paste in the Bulk Add Questions field in the survey builder.
Crowdtech
Crowdtech native format (.ctsurvey): contains all question and answer content. Upload via Download Center > Import Survey. Select elements to include before importing.
SurveyVista
CSV file (.csv): formatted for SurveyVista's Survey Import interface. Upload via Surveys tab > Import Survey > Upload Files.
