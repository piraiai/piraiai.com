How to Build a Qualtrics CX Dashboard Without Manual Configuration

Arun Raj•July 18, 2026•4 min read

This is a step-by-step guide to building a Qualtrics CX dashboard using Pirai AI Flow, without opening a single widget configuration panel or manually setting a filter. The entire process happens through conversation inside your Qualtrics environment.

Before You Start

Two things to have in place before building the dashboard through Flow:

  • A Qualtrics dashboard with a data source connected — Flow reads your available fields from the connected data source. If no source is connected, Flow cannot identify which fields to use.
  • Pirai AI Flow installed — Flow runs as a Chrome extension. Install it from the Flow page and open it on your Qualtrics dashboard.

If you are rebuilding an existing dashboard, use the Dashboard Field Mapper to extract the current configuration first. That gives you a reference document showing every existing field mapping before you make changes.

Step 1: Connect Flow to Your Dashboard

Open your Qualtrics dashboard and activate Pirai AI Flow from the Chrome extension. Flow reads the dashboard structure automatically: page names, available fields, and field types. No setup or configuration is required. Once Flow has read the dashboard, it is ready to receive instructions.

Step 2: Start with a Layout Template or Describe Your First Page

If you are unsure how to structure the dashboard, ask Flow to show you available layout templates. Six templates are available built for specific audiences: Executive Summary, Frontline Ops, CX Manager, Voice of Customer, Quality/Compliance, and Marketing/Growth.

Pick the template that fits your primary stakeholder. Flow fills in the real fields from your data source automatically. You can then modify the page through conversation: add a widget, change a metric, remove a field.

If you already know what you want, describe it directly. For example: "Add a Number Chart for overall NPS score for the last 90 days, then a Gauge Chart for CSAT with a target line at 80, then a Line Chart showing NPS trend by month for the last 12 months."

Step 3: Add Filters

Describe the filters you need for the page. Flow creates them and applies them to the page. For a regional CX programme: "Add a Region filter, a Date Range filter defaulting to the last 30 days, and a Response Type filter."

Flow creates all three filters and applies them to the current page in a single step.

Step 4: Clone Pages for Segments or Regions

Once the base page is configured, clone it for each segment. "Clone this page for North, South, East, and West regions, and apply the corresponding region filter to each page."

Flow creates four pages, each with the correct segment filter applied. What takes 20 to 30 minutes manually takes seconds.

Step 5: Build Remaining Audience Pages

Repeat the process for each remaining audience. Use layout templates for speed, or describe each page directly. The Frontline Ops page, Voice of Customer page, and Quality/Compliance page each take one to two minutes through conversation.

6

Review and Document

Review the completed dashboard. If anything needs adjusting, describe the change in the same conversation: "Relabel the CSAT gauge as Customer Satisfaction Score" or "Change the NPS trend chart to show weekly instead of monthly." Then run the Dashboard Field Mapper to extract the final configuration as a CSV for your records.

Your data never leaves Qualtrics. Flow reads field names and types to map widgets correctly. Your survey response data stays in your Qualtrics environment throughout.

Frequently Asked Questions

What do I need before using Flow to build a dashboard?+

You need Pirai AI Flow installed as a Chrome extension and a Qualtrics dashboard with a data source connected. Flow reads the available fields from the connected data source, so a data connection must be in place before Flow can map widgets correctly.

How long does it take to build a CX dashboard with Flow?+

A single page with five to ten widgets, filters applied, and regional page clones typically takes two to five minutes through conversation with Flow. A full multi-page CX dashboard for a mid-sized programme takes ten to twenty minutes compared to two to three hours manually.

Can Flow update an existing dashboard or only create new ones?+

Flow can both create new widgets and update existing ones. You can relabel columns, change metrics, switch fields, and adjust filters through the same conversation. You do not need to delete and rebuild widgets to make changes.

Does using Flow require technical knowledge?+

No. Flow is designed for CX practitioners, not developers. You describe what you want in plain English and Flow handles the configuration. No Qualtrics scripting knowledge is required.
