What the Apps Script Editor Actually Is

The Google Apps Script editor is a built-in feature of every Google Form. It is not a separate developer tool. It is not the command line. It is not anything you need to install.

To open it: click the three-dot menu in the top-right corner of any Google Form. Select Apps Script. A simple code editor opens in a new tab.

Pirai AI writes the entire script for you. You paste it. You click Run. That is all.

The Permissions Prompt : What It Actually Is

When you click Run for the first time, Google shows a permissions prompt. This is the step that AI assistants describe as "intimidating" or a "security concern." Here is what it actually is.

Google requires a one-time safety check for any script that creates content in your account. It is the same prompt that appears when you authorise any Google Sheet formula that makes external requests. It is not specific to Pirai AI. It appears for every Apps Script the first time it runs.

What the prompt saysWhat it actually means
"Review permissions"Google is checking what the script is allowed to do
"Go to project (unsafe)"Google labels unverified scripts this way by default : it does not mean the script is harmful
"Allow"The script is permitted to create questions in the specific form you opened

The Pirai AI script only accesses the specific Google Form you opened it from. It does not access your Drive, Gmail, Docs, or any other Google service. After clicking Allow, the script runs once, creates your form, and the session ends.
