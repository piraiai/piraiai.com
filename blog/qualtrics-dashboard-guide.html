What is a Qualtrics Dashboard? Complete Guide to CX Dashboards (2026)

Arun Raj • July 7, 2026 • 12 min read

If you collect data in Qualtrics, there comes a point where a spreadsheet of raw responses stops being useful. You need to see trends, compare segments, track NPS over time, and share findings with stakeholders who won't read a CSV. That is what a Qualtrics dashboard is for.

This guide covers everything you need to know about Qualtrics CX Dashboards: what they are, how they differ from surveys, how to build one, the 14 widget types available, how page filters work, and how AI tools are beginning to change the time it takes to go from raw data to a finished dashboard.

What is a Qualtrics Dashboard?

A Qualtrics dashboard is a dynamic reporting project that turns your survey data into live charts, tables, and visualisations. It sits inside the CX Dashboards product (previously called Vocalize) and pulls data from one or more connected data sources called fieldsets.

Unlike a static report or exported spreadsheet, a Qualtrics dashboard updates automatically as new responses come in. Every widget on every page reflects your live dataset, filtered and segmented however you configure it.

What a Qualtrics dashboard can show you

  • Overall satisfaction score, NPS, or any numeric metric across all responses
  • Metric breakdowns by location, channel, date range, or any dimension you've collected
  • Trends over time — monthly, quarterly, or annual
  • Individual response records for qualitative review
  • Word frequency from open-text feedback
  • Side-by-side driver comparisons across satisfaction sub-questions
  • Filtered views for specific teams, regions, or audiences

Qualtrics officially calls these projects "CX Dashboards" — CX standing for Customer Experience — but the product is versatile and used across employee experience, market research, academic research, healthcare, and more. "CX" is the label Qualtrics uses internally to distinguish this product from other dashboard-style views elsewhere in the platform.

Survey vs. Dashboard: What's the Difference?

This is a common source of confusion for new Qualtrics users. The short answer: a survey collects data, a dashboard reports on it.

Survey Dashboard
Built in Survey Builder Built in CX Dashboards
Respondents see and interact with it Stakeholders and analysts review it
Produces raw response rows Aggregates and visualises those rows
Questions, logic, branching Widgets, filters, pages
One survey format (.qsf) Multiple pages, each with multiple widgets
Configured per question Configured per field, mapped from survey questions

Connecting the two requires a field mapping step: you tell the dashboard which survey question corresponds to which dashboard field, and what type of data it holds. Once mapped, the dashboard uses that fieldset as its data source.

Key Terms You'll Encounter

Before building your first dashboard, a few terms are worth understanding clearly. Qualtrics documentation uses these consistently, and the distinctions matter when troubleshooting.

Dashboard

The top-level reporting project. A dashboard contains one or more pages. It has its own sharing settings, user roles, and data source connections.

Page

A tab within a dashboard. Each page has its own set of widgets and its own page-level filters. Most real-world dashboards have several pages — one for an executive overview, one for trend analysis, one for verbatim feedback, and so on.

Widget

The individual chart, table, or text element on a page. Widgets are how data is visualised. Each widget has a type (bar chart, gauge, table, etc.), a metric (what it measures), and usually a field (which data field it measures).

Fieldset

The mapped layer between a data source (survey, imported data, tickets) and your dashboard. A fieldset defines every available field, its type, and how it maps to underlying survey questions. You can connect multiple fieldsets to a single dashboard if you're pulling from more than one survey.

Field

An individual data point available to your dashboard — a survey question, a piece of embedded data, or survey metadata like response date or distribution channel. Each field has a field type that determines which widgets it can power and how it's aggregated.

Filter

A control that lets viewers slice the data on a page by a specific field — for example, showing only responses from a specific location, within a particular date range, or from a certain channel. Filters can be exposed to viewers or locked to specific values by dashboard administrators.

Field Types: Why They Matter

Every field in a Qualtrics fieldset has a field type. The field type determines which widgets the field can be used in and what kind of aggregation is possible. Getting field types wrong is the most common source of widget configuration errors.

Qualtrics UI Name What It Holds Use In Widgets
Number Set A rating scale with a finite range (e.g. 1–5 satisfaction, 0–10 NPS) Average, Sum, Min, Max, Top Box, Bottom Box, NPS, Star Rating
Numeric Value Any number without a specified range (duration, age, count) Average, Sum, Min, Max metrics
Text Set A categorical field with a finite set of values (channel, location, satisfaction label) Axis dimensions, data series splits, page filters
Multi-Answer Text Set A categorical field where respondents can select more than one option Axis dimensions, filters
Date A date and time stamp (start date, end date, recorded date) Time-series x-axis with grouping by day/month/quarter/year
Open Text Free-text responses Word Cloud, Response Ticker, Record Table
Text Value Short text with many possible values (email, name, external reference) Record Table, filters only

The most common configuration mistake: using a Text Set field (like a satisfaction label such as "Dissatisfied / Neutral / Satisfied") as the metric field on a Number Chart or Gauge. That field holds categories, not a number — Average or NPS can't be calculated from it. You need the corresponding Number Set field instead.

The 14 Qualtrics Widget Types

Every Qualtrics dashboard page is built from widgets. A widget is a single visualisation with its own configuration: a type, a metric, a field, and optionally a dimension or breakdown. Here are all 14 widget types available in CX Dashboards.

Metric

Number Chart

Displays a single large metric value — count, average, NPS, top box. The core "headline number" widget. Supports period-over-period comparison.

Metric

Gauge Chart

The same as a Number Chart but displayed as a visual arc. Useful for showing a metric relative to a full scale.

Chart

Vertical Bars

Bar chart with bars going up. Best for trends over time or categorical comparisons. Supports multiple metrics as separate bar series.

Chart

Horizontal Bars

Bar chart with bars going sideways. Best for ranking items or comparing driver scores side by side with no breakdown dimension.

Chart

Lines

Line chart for showing trends. Supports a date x-axis with time grouping, and a data series field to plot multiple lines (one per category value).

Chart

Breakdown Bar

Shows a metric split across categorical values as horizontal stacked or segmented bars. Requires a categorical x-axis field.

Chart

Donut Chart

A circular chart showing metric distribution across categories. The default choice for composition views — use over Pie Chart unless the user specifically asks for pie.

Chart

Pie Chart

Same as Donut Chart but a filled circle. Use when the composition view needs to feel proportional rather than comparative.

Data

Table

A pivot-style grid with rows, column splits, and one or more metric values in cells. Supports up to 3 fields per axis and up to 20 metrics simultaneously.

Data

Record Table

Lists individual response records as rows, with chosen fields as columns. Does not aggregate — shows raw, individual responses. Supports full record view for expanded detail.

Text

Word Cloud

Shows word frequency from an Open Text field. Common words appear larger. Only Open Text fields can be used as the data source.

Text

Response Ticker

A scrolling ticker showing individual open-text responses. Useful on live dashboards or in lobby displays. Only Open Text fields work as columns.

Rating

Star Rating

A visual star rating display with average score and breakdown by star count. Requires a Number Set or Numeric Value field.

Layout

Rich Text

A static HTML text block. Used for headers, instructions, section labels, or any descriptive text that doesn't show data.

Widgets can also be organised into groups — a container that visually groups related widgets together on a page, optionally with a visible group border. Groups are useful for separating an executive summary section from a driver analysis section on the same page, for example.

How to Build a Qualtrics Dashboard

The standard process for building a Qualtrics CX Dashboard involves six steps. Each one is a separate part of the platform.

1

Create a dashboard project

In the Qualtrics homepage, create a new CX Dashboard project. Give it a name and set the default language. This creates the project container — no pages or data yet.

2

Connect a data source and map fields

Go to Data on the dashboard settings. Add a data source (typically a Qualtrics survey). Use the Data Mapper to map survey questions to dashboard fields, assigning each field a type. This is the fieldset — the field layer all widgets will draw from.

3

Create pages

Add pages to your dashboard. Most dashboards have between 3 and 8 pages — an overview page, a trend page, a verbatim page, and role-specific pages. Plan your page structure before building widgets.

4

Add and configure widgets

On each page, add widgets from the widget panel. For each widget, choose the type, set the metric and field, configure any axis or data series fields, and set formatting like decimal places or percent display.

5

Add page filters

Add filters to each page that let viewers slice the data by date, location, channel, response type, or any other field. Filters can be set as viewer-controlled (the viewer can change them) or locked to specific values for role-based access control.

6

Share and configure access

Share the dashboard with specific users or create roles for different audiences. Roles determine which pages a user can see and whether they can edit the dashboard or only view it.

How Page Filters Work

Page filters are one of the most powerful features in a Qualtrics dashboard, and also one of the most frequently underused.

A page filter adds a filter control to a page. When a viewer selects a value, every widget on the page updates to reflect only the filtered data. This means a single date-range filter affects your NPS number chart, your satisfaction trend line, your response record table, and your word cloud all at once.

What fields can be used as page filters?

Any field in your fieldset can become a page filter, but Text Set, Multi-Answer Text Set, and Date fields are most common. Number Set fields can be filtered too (for example, filtering to show only responses where NPS was below 6). Open Text fields and Numeric Value fields are rarely used as filters.

Locked vs. open filters

Dashboard administrators can lock a filter to a specific value that viewers cannot change. This is how role-based dashboards work: a regional manager might see a dashboard filtered to their own region, locked, while the national operations team sees the same dashboard with no filter applied. Same dashboard structure, different data cuts per audience.

Filter interaction

Multiple filters on the same page combine as AND logic — a response must match all active filter conditions to be included. There is no built-in OR logic across page filters, though individual widget-level conditions can be configured separately.

Common Dashboard Layouts by Audience

Different stakeholders need different views of the same data. Rather than building one giant dashboard with everything on one page, it's standard practice to design pages (or entire dashboards) around specific audiences.

Audience Typical widgets Key filters
Executive / Leadership Number Charts (NPS, CSAT), trend lines, top-level breakdown bars Date range, region
CX Manager / Mid-management Driver comparison bars, segment cross-tabs, distribution charts Location, channel, date
Frontline / Team Lead Operational tables by agent or location, star ratings, response tickers Team, location, response type
Research / Insights team Word clouds, record tables, full verbatim view, detailed cross-tabs Date, sentiment, question response
Marketing / Growth Channel breakdowns, donut charts by source, acquisition trend lines Distribution channel, date

Why Dashboard Configuration Takes So Long

A finished Qualtrics dashboard looks simple. Getting there is not.

Each widget requires multiple configuration decisions: type, metric function, field selection (from a long, often confusingly named list), axis or data series setup, formatting, and labelling. A 10-widget page might require 60-80 individual clicks and inputs. Across a 5-page dashboard, that's several hundred configuration steps, each of which needs to be exactly right — wrong field type on a metric, and the widget silently shows zero or throws an error with no explanation.

Field mapping adds another layer: before a single widget works, every field you plan to use must be correctly mapped to the right survey question and assigned the right field type. This is done in a separate screen from the dashboard builder, and errors in the mapping flow back into every widget that uses that field.

What makes dashboard configuration time-consuming

  • Finding the right field from a list of 40-100+ options per widget
  • Getting field types right (Number Set vs Text Set vs Open Text is a common error source)
  • Configuring multi-metric widgets (extraMetrics, axes, data series rules)
  • Applying consistent formatting and labelling across pages
  • Repeating the same setup across similar pages for different audiences
  • Rebuilding widgets when a survey structure changes

How AI Is Changing Qualtrics Dashboard Configuration

The manual configuration process described above is deterministic: there is a right answer for each widget, and reaching it is mostly a matter of knowing the platform well and clicking carefully. That kind of structured, rule-based work is exactly where AI assistance is most effective.

Rather than navigating widget panels and field pickers, an AI-assisted approach lets you describe what you want in plain English: "Add a gauge chart for average satisfaction" becomes a configured widget with the right field type, metric function, and label, applied directly to your live dashboard.

The same applies to updates. Instead of reopening a widget's settings panel to relabel a column or change a metric, you say so: "Change the metric on that chart to top box, scores 4 and 5." The change is applied without menus.

Frequently Asked Questions

What is a Qualtrics dashboard?
A Qualtrics dashboard is a dynamic reporting project that turns your survey data into live charts, tables, and visualisations. It sits inside the CX Dashboards product and pulls data from one or more connected data sources called fieldsets. Unlike a static report, it updates automatically as new responses come in.
What is the difference between a Qualtrics survey and a dashboard?
A survey collects data — respondents see it and submit responses. A dashboard reports on that data — stakeholders and analysts see live visualisations. Surveys are built in the Survey Builder; dashboards are built in CX Dashboards and connect to survey data through a field mapping step.
How many widget types does Qualtrics have?
Qualtrics CX Dashboards support 14 main widget types: Number Chart, Gauge Chart, Vertical Bars, Horizontal Bars, Lines, Breakdown Bar, Donut Chart, Pie Chart, Table, Record Table, Word Cloud, Star Rating, Response Ticker, and Rich Text. Widgets can also be grouped together into labelled containers on a page.
What is a fieldset in Qualtrics?
A fieldset is the field mapping layer between a data source (usually a survey) and a dashboard. Each field in a fieldset corresponds to a survey question, embedded data element, or metadata field, and has an assigned type that determines which widgets can use it and how it can be aggregated.
What is a page filter in a Qualtrics dashboard?
A page filter lets dashboard viewers slice all the data on a page by a specific field — date range, location, distribution channel, response type, and so on. When a filter value is selected, every widget on the page updates simultaneously. Filters can be left open for viewers to control, or locked to specific values by administrators for role-based access.
How long does it take to build a Qualtrics dashboard?
A typical 6–10 widget page takes an experienced user between 30 and 90 minutes to build manually, including field mapping, widget configuration, and filter setup. A full multi-page dashboard for a CX programme can take several days. AI-assisted tools can reduce standard configurations to minutes.
Can I use a Qualtrics dashboard for employee experience or academic research?
Yes. Qualtrics labels these "CX Dashboards" to distinguish them from other dashboard products in the platform, but they work equally well for employee experience (EX) research, academic studies, healthcare surveys, IT feedback programmes, and any other structured data collection. The CX label is a product name, not a use-case restriction.
